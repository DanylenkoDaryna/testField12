Сборка позволяет работать с Java WEB проектом в IntelliJ IDEA Community Edition.

*********************************************************************

1. Открыть проект в IntelliJ IDEA (это обычный Java проект для IDEA).

2. Установить правильное значение переменной среды JAVA_HOME в файле

	setenv.bat (для Windows OS)
	setenv.sh  (для Linix/MacOS)

3. Запустить Tomcat

	tomcat-start.bat (для Windows OS)
	tomcat-start.sh  (для Linix/MacOS)

Приложение будет доступно по адресу http://localhost:8080/app
http://localhost:8080/app

*********************************************************************

Tomcat автоматически учитывает (с небольшой временной задержкой) изменения в каталоге webapp
(это корень WEB приложения).
Пути в проекте сконфигурированы таким образом, что результат компиляции исходных файлов
попадает в каталог
	
	webapp/WEB-INF/classes

поэтому после компилияции файлов в IDEA, Tomcat должен подхватить изменения.


*********************************************************************

В некоторых случаях потребуется сделать редеплой с помощью скрипта

	redeploy.bat (для Windows OS)
	redeploy.sh  (для Linix/MacOS)

Редеплой необходимо делать после создания новых слушателей, фильтров, сервлетов.

Если менять маппинг существующих сервлетов, то Tomcat подхватит эти изменения
автоматически без редеплоя.
